# Revenue Cycle Visibility Tracker

## Purpose

This tool supports the Healthcare Operations Proof System by turning the Revenue Cycle Visibility Proof case study into a practical tracker.

It is designed to show how revenue cycle teams can connect claims, denials, root causes, workqueue priority, ownership, next actions, and operational metrics in one structured workflow view.

## Workbook File

Upload the Excel workbook to this folder:

```text
/tools/Revenue_Cycle_Visibility_Tracker_Kori_Pickle.xlsx
```

## Workbook Tabs

| Tab | Purpose |
|---|---|
| Dashboard | Summarizes key revenue cycle visibility metrics |
| Claim Tracker | Tracks simulated claims, root causes, owners, next actions, due dates, A/R days, and priority tiers |
| Denial Root Causes | Maps denial categories to workflow sources, downstream risks, prevention actions, and metrics |
| Workqueue Priority | Defines Tier 1, Tier 2, and Tier 3 prioritization logic |
| Metric Definitions | Explains each metric, formula, operational purpose, owner, and target direction |

## Core KPIs

- Total Claims Tracked
- Denial Rate
- Clean Claim Rate
- Average Days in A/R
- High-Risk Accounts
- Authorization-Related Denials
- Eligibility-Related Denials
- Rework Volume

## Operational Value

This tracker demonstrates practical healthcare operations thinking by showing how workflow visibility can help identify:

- denial root causes
- eligibility breakdowns
- authorization delays
- A/R risk
- workqueue ownership gaps
- next-action delays
- recurring rework sources
- high-priority account risk
